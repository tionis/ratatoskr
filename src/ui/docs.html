Offline-First: This library supports offline document creation. Documents are stored in IndexedDB and sync automatically when connectivity is restored.

Quick Start

The fastest way to get started is with a direct ESM import from this server (no build step required):

<script type="module">
import { RatatoskrClient } from '{{SERVER_URL}}/ratatoskr-client.js';

const client = new RatatoskrClient({
  serverUrl: '{{SERVER_URL}}'
});

// Login via popup
const user = await client.login();
console.log('Logged in as:', user.name);

// Get automerge repo for real-time sync
const repo = client.getRepo();

// Create a document
const { id } = await client.createDocument({
  type: 'notes' // Optional type
});
console.log('Created document:', id);

// List your documents
const { owned, accessible } = await client.listDocuments();
console.log('My documents:', owned);
</script>

Installation

Option 1: Direct ESM Import (Recommended for HTML tools)

No installation needed. Import directly from this server:

import { RatatoskrClient } from '{{SERVER_URL}}/ratatoskr-client.js';

This bundle includes all dependencies (including automerge) so there's nothing else to install.

Option 2: npm/bun/yarn

npm install ratatoskr-client @automerge/automerge-repo
import { RatatoskrClient } from 'ratatoskr-client';
Peer Dependency: This library requires @automerge/automerge-repo as a peer dependency for real-time sync features. If you only need REST API access, you can skip it.

API Reference

RatatoskrClient

The main client class for interacting with a Ratatoskr server.

Constructor

new RatatoskrClient(options: RatatoskrClientOptions)
Option Type Required Default Description
serverUrl string Required - Base URL of the Ratatoskr server
tokenStorageKey string Optional "ratatoskr:token" localStorage key for token persistence
enableOfflineSupport boolean Optional true Enable offline-first document creation with IndexedDB storage

Properties

  • userDocHandle: Handle to the user's profile document (synced on login).
  • appDocHandle: Handle to the application's root document (synced if appId is provided).

Authentication Methods

login(): Promise<User>

Opens a popup for OIDC authentication. Returns user info on success. Requires network connectivity.

const user = await client.login();
// { id: "alice", name: "Alice Smith", email: "alice@example.com" }

logout(): void

Clears stored credentials and disconnects.

client.logout();

isAuthenticated(): boolean

Check if user has a stored token (doesn't validate the token).

if (client.isAuthenticated()) {
  // Token exists (may still be expired)
}

hasStoredCredentials(): boolean

Check if both token and user info are cached. Useful for "Welcome back" UI even when offline.

if (client.hasStoredCredentials()) {
  console.log(`Welcome back, ${client.getUser()?.name}!`);
}

getUser(): User | null

Get the current user object. When offline with stored credentials, returns the cached user.

fetchUserInfo(): Promise<User>

Fetch/validate current user info from the server. Updates the cached user info.

try {
  const user = await client.fetchUserInfo();
  console.log('Token valid for:', user.name);
} catch {
  console.log('Token expired');
}

validateToken(): Promise<boolean>

Validate the stored token by fetching user info. Returns true if valid, false if expired/invalid.

if (client.isAuthenticated()) {
  const isValid = await client.validateToken();
  if (!isValid) {
    // Token expired, prompt re-login
  }
}

Automerge Integration

getRepo(): Repo

Get or create the automerge-repo instance connected to the server.

const repo = client.getRepo();

// Find an existing document
const handle = repo.find('2k5...'); // Use standard Automerge ID
await handle.whenReady();
const doc = handle.docSync();

// Create a new document locally
const newHandle = repo.create();
newHandle.change(doc => {
  doc.title = 'My Document';
  doc.items = [];
});

disconnect(): void

Disconnect from the server and clean up the repo.

Document Management

createDocument(request): Promise<DocumentMetadata>

Create a new document on the server.

const doc = await client.createDocument({
  type: 'notes',                // Optional
  acl: [                        // Optional
    { principal: 'bob', permission: 'write' },
    { principal: 'public', permission: 'read' }
  ],
  expiresAt: '2025-12-31'       // Optional
});

Document ID Types:

TypeFormatDescription
StandardBase58 (e.g. 2k5...)Regular persistent document with full ACL support. Default type.
App DocBase58Private per-app root document. Managed via appId.
Ephemeraleph:<id>Relay-only document for temporary sessions. No persistence.

listDocuments(): Promise<ListDocumentsResponse>

List documents you own or have access to.

const { owned, accessible } = await client.listDocuments();
console.log('My documents:', owned);
console.log('Shared with me:', accessible);

getDocument(id): Promise<DocumentMetadata>

Get metadata for a specific document.

const doc = await client.getDocument('2k5...');
console.log('Owner:', doc.owner, 'Size:', doc.size);

deleteDocument(id): Promise<void>

Delete a document (owner only).

await client.deleteDocument('2k5...');

createEphemeralDocument(id?): DocHandle<any>

Create (or join) an ephemeral document. These are relay-only and not persisted.

// Create new session
const handle = client.createEphemeralDocument();
console.log('Session ID:', handle.documentId); // "eph:..."

// Join existing session
const session = client.createEphemeralDocument('eph:my-session');

Access Control

getDocumentACL(id): Promise<ACLEntry[]>

Get the access control list for a document.

const acl = await client.getDocumentACL('2k5...');
// [{ principal: 'bob', permission: 'write' }]

setDocumentACL(id, acl): Promise<void>

Update a document's ACL (owner only).

await client.setDocumentACL('2k5...', [
  { principal: 'bob', permission: 'write' },
  { principal: 'charlie', permission: 'read' },
  { principal: 'public', permission: 'read' }
]);

ACL Principal Types:

PrincipalDescription
User IDGrant access to specific user
Document IDInherit ACL from another document (Recursive ACL)
"public"Grant access to everyone

Permissions:

PermissionAllows
"read"Read document content
"write"Read and write document content

API Tokens

createApiToken(name, scopes?, expiresAt?): Promise<{token, id}>

Create an API token for CLI or programmatic access.

const { token, id } = await client.createApiToken(
  'my-cli-tool',
  ['read', 'write'],       // Optional scopes
  '2026-01-01T00:00:00Z'   // Optional expiration
);
console.log('Save this token:', token);  // Only shown once!

listApiTokens(): Promise<ApiToken[]>

List all your API tokens (token values not returned).

const tokens = await client.listApiTokens();
for (const t of tokens) {
  console.log(`${t.name} (${t.id}) - last used: ${t.lastUsedAt}`);
}

deleteApiToken(id): Promise<void>

Revoke an API token.

await client.deleteApiToken('token-id-here');

Offline-First Document Creation

Documents can be created and edited offline. They'll be registered on the server when connectivity and authentication are restored.

createDocumentOffline<T>(initialValue, options?): Promise<string>

Create a document that works offline. Returns the document ID immediately.

// Create document offline - works even without network
const docId = await client.createDocumentOffline(
  { title: 'My Notes', items: [] },
  { type: 'notes' }
);

// Document is usable immediately via automerge
const handle = client.getRepo().find(docId);
handle.change(doc => {
  doc.items.push('First item');
});

// When online + authenticated, document auto-registers on server
Note: Documents created offline are private (no ACLs) until you're online. You can set ACLs after the document syncs using setDocumentACL().

getDocumentSyncStatus(documentId): Promise<DocumentStatusEntry | undefined>

Get the sync status of a document.

const status = await client.getDocumentSyncStatus(docId);
if (status) {
  console.log('Status:', status.status); // 'local' | 'syncing' | 'synced'
  console.log('Registered on server:', status.serverRegistered);
}

getConnectivityState(): ConnectivityState

Get the current connectivity state: 'online', 'offline', or 'connecting'.

processPendingOperations(): Promise<{processed, failed}>

Force processing of pending sync operations. Useful after login.

await client.login();
const result = await client.processPendingOperations();
console.log(`Synced ${result.processed} documents`);

onSyncEvent(listener): () => void

Subscribe to sync events. Returns an unsubscribe function.

const unsubscribe = client.onSyncEvent((event) => {
  switch (event.type) {
    case 'connectivity:changed':
      console.log('Connectivity:', event.connectivity);
      break;
    case 'document:status-changed':
      console.log(`Document ${event.documentId}: ${event.status}`);
      break;
    case 'sync:completed':
      console.log(`Synced ${event.processed} documents`);
      break;
    case 'auth:token-expired':
      console.log('Token expired, please log in again');
      break;
  }
});

Event Types:

EventDescription
connectivity:changedNetwork/server connection state changed
document:status-changedA document's sync status changed
sync:startedSync processing started
sync:completedSync processing completed
auth:requiredAuthentication needed to continue syncing
auth:token-expiredToken is invalid/expired

destroy(): void

Cleanup all resources including IndexedDB connections.

Blob Storage

Ratatoskr provides content-addressable blob storage with automatic deduplication. Blobs are identified by their SHA-256 hash and can be shared across users.

Uploading Blobs

uploadBlob(data, options?): Promise<BlobInfo>

Upload a blob to the server. Supports chunked uploads for large files with progress tracking.

// Upload a file with progress tracking
const file = document.querySelector('input[type="file"]').files[0];
const blob = await client.uploadBlob(file, {
  onProgress: (progress) => {
    console.log(`${progress.phase}: ${progress.bytesProcessed}/${progress.totalBytes}`);
    // Phases: "hashing" -> "checking" -> "uploading" -> "complete"
  }
});
console.log('Uploaded:', blob.hash);

// Upload raw data
const data = new Uint8Array([1, 2, 3, 4]);
const result = await client.uploadBlob(data, {
  mimeType: 'application/octet-stream'
});

Progress Phases:

PhaseDescription
hashingComputing SHA-256 hash locally
checkingChecking if blob already exists (deduplication)
uploadingUploading chunks to server
completeUpload finished

Downloading Blobs

downloadBlob(hash): Promise<Uint8Array>

Download a blob by its hash.

const data = await client.downloadBlob('abc123...');
// Use the data...

getBlobUrl(hash): string

Get the direct URL for a blob. Useful for <img src="..."> or direct links.

const url = client.getBlobUrl('abc123...');
document.querySelector('img').src = url;

getBlobInfo(hash): Promise<BlobInfo | null>

Get blob metadata without downloading the content.

const info = await client.getBlobInfo('abc123...');
if (info) {
  console.log(`Size: ${info.size}, Type: ${info.mimeType}`);
}

Blob Claims

Blobs use a claims-based lifecycle. Each user who needs a blob claims it, which counts towards their storage quota. When all claims are released, the blob is cleaned up after a 24-hour grace period.

claimBlob(hash): Promise<BlobInfo>

Claim an existing blob by its hash. Useful when you know a blob's hash and want to ensure it's not deleted.

const blob = await client.claimBlob('abc123...');

releaseBlobClaim(hash): Promise<void>

Release your claim on a blob.

await client.releaseBlobClaim('abc123...');

listClaimedBlobs(options?): Promise<ListBlobsResponse>

List all blobs you've claimed with quota information.

const { blobs, total, quotaUsed, quotaLimit } = await client.listClaimedBlobs({
  limit: 20,
  offset: 0
});

console.log(`Using ${quotaUsed} of ${quotaLimit} bytes`);

Document Blob Claims

Blobs can be attached to documents. When the document is deleted, its blob claims are automatically released.

uploadBlobToDocument(documentId, data, options?): Promise<BlobInfo>

Upload a blob and immediately attach it to a document.

const blob = await client.uploadBlobToDocument('2k5...', file, {
  onProgress: (p) => console.log(p.phase)
});

addDocumentBlobClaim(documentId, hash): Promise<void>

Attach an existing blob to a document.

await client.addDocumentBlobClaim('2k5...', 'abc123...');

removeDocumentBlobClaim(documentId, hash): Promise<void>

Remove a blob from a document.

await client.removeDocumentBlobClaim('2k5...', 'abc123...');

listDocumentBlobs(documentId): Promise<DocumentBlobsResponse>

List all blobs attached to a document.

const { blobs, totalSize } = await client.listDocumentBlobs('2k5...');
console.log(`Document has ${blobs.length} blobs (${totalSize} bytes)`);

  quotaLimit: number;

  blobs: BlobInfo[];
  totalSize: number;

type DocumentSyncStatus = 'local' | 'syncing' | 'synced';

  serverRegistered: boolean;
  createdAt: string;

type SyncEventType =
  | 'sync:started'

  | 'sync:error'

  | 'connectivity:changed'

  type: SyncEventType;

Examples

Single-File HTML Tool

<!DOCTYPE html>
<html>
<head>
  <title>My Collaborative Tool</title>
</head>
<body>
  <button id="login">Login</button>
  <button id="create" disabled>Create Document</button>
  <div id="output"></div>

  <script type="module">
    import { RatatoskrClient } from '{{SERVER_URL}}/ratatoskr-client.js';

    const client = new RatatoskrClient({
      serverUrl: '{{SERVER_URL}}'
    });

    // Check for existing session
    if (client.isAuthenticated()) {
      try {
        await client.fetchUserInfo();
        enableApp();
      } catch {
        client.logout();
      }
    }

    document.getElementById('login').onclick = async () => {
      try {
        const user = await client.login();
        document.getElementById('output').textContent = `Hello, ${user.name}!`;
        enableApp();
      } catch (err) {
        alert('Login failed: ' + err.message);
      }
    };

    document.getElementById('create').onclick = async () => {
      // Create document (let server generate ID)
      const doc = await client.createDocument({
        type: 'notes'
      });
      console.log('Created:', doc);
    };

    function enableApp() {
      document.getElementById('create').disabled = false;
    }
  </script>
</body>
</html>

Real-Time Collaboration with Automerge

import { RatatoskrClient } from '{{SERVER_URL}}/ratatoskr-client.js';

const client = new RatatoskrClient({
  serverUrl: '{{SERVER_URL}}'
});

await client.login();

// Create document and get ID
const { id } = await client.createDocument({ type: 'counter' });

// Get automerge repo
const repo = client.getRepo();

// Find the document
const handle = repo.find(id);

// Make changes that sync in real-time
handle.change(doc => {
  doc.count = 0;
});

// Listen for remote changes
handle.on('change', ({ doc }) => {
  console.log('Count is now:', doc.count);
});

// Increment counter
function increment() {
  handle.change(doc => {
    doc.count++;
  });
}

Share a Document

// Create a document
const { id } = await client.createDocument({
  type: 'notes'
});

// Share with specific user (write access)
await client.setDocumentACL(id, [

// Make it publicly readable
await client.setDocumentACL(id, [
  { principal: 'bob', permission: 'write' },

]);

CLI Tool with API Token

// First, create a token via the web UI or API
// Then use it directly with fetch:

const RATATOSKR_URL = '{{SERVER_URL}}';
const API_TOKEN = process.env.RATATOSKR_TOKEN;

async function listMyDocuments() {
  const response = await fetch(`${RATATOSKR_URL}/api/v1/documents`, {
    headers: {
      'Authorization': `Bearer ${API_TOKEN}`
    }
  });
  return response.json();
}

const { owned, accessible } = await listMyDocuments();
console.log('My documents:', owned.map(d => d.id));

Error Handling

All async methods throw errors on failure. Errors include descriptive messages:

try {
  await client.login();
} catch (err) {
  if (err.message.includes('popup')) {
    // Popup was blocked or closed by user
    alert('Please allow popups for this site');
  } else if (err.message.includes('timeout')) {
    // Authentication took too long (5 minute limit)
    alert('Authentication timed out, please try again');
  } else if (err.message.includes('cancelled')) {
    // User closed the popup
    console.log('User cancelled login');
  }
}

try {
  await client.createDocument({ id: 'doc:test' });
} catch (err) {
  // err.message contains server error details
  // e.g., "Document already exists", "Quota exceeded"
  console.error('Failed to create document:', err.message);
}

try {
  await client.setDocumentACL('2k5...', [...]);
} catch (err) {
  // e.g., "Not authorized" (not the owner)
  console.error('Failed to update ACL:', err.message);
}
